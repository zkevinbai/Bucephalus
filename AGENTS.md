# Agent Instructions

This document provides guidance for AI agents and assistants working on the Bucephalus codebase.

## Codebase Overview

Bucephalus is a personal portfolio website built with React, Vite, and Tailwind CSS. It's a static single-page application deployed to GitHub Pages.

## Key Principles

1. **Keep it simple**: This is a personal portfolio site, not an enterprise application
2. **Maintain consistency**: Follow existing patterns and the design system
3. **Preserve functionality**: Never break existing features when making changes
4. **Clean code**: Remove unused code, dependencies, and files
5. **Documentation**: Update relevant docs when making architectural changes

## Architecture

- **Framework**: React 18 with functional components and hooks
- **Routing**: React Router DOM (BrowserRouter)
- **Styling**: Tailwind CSS utility classes
- **Build**: Vite (builds to `docs/` folder)
- **Deployment**: GitHub Pages (serves from `docs/` folder)

See [SYSTEM_DESIGN.md](./SYSTEM_DESIGN.md) for detailed architecture.

## Design System

All styling must follow the design system documented in [KEVIN_DESIGN_SYSTEM.md](./KEVIN_DESIGN_SYSTEM.md):

- Use Tailwind utility classes
- Follow color palette and typography guidelines
- Maintain responsive design patterns
- Use consistent spacing and component patterns

## File Organization

```
src/
├── components/      # Shared UI components (Header, Breadcrumbs)
├── features/        # Feature modules (Portfolio, Blog, ChineseZodiac)
└── assets/         # Static assets (images, logos)
```

**Rule**: Keep features self-contained. Shared components go in `components/`, feature-specific components stay in their feature folder.

## Common Patterns

### Adding a New Route

1. Add route to `src/App.jsx`:
```javascript
<Route path="/new-route" element={<NewComponent />} />
```

2. Add navigation link to `src/components/Header.jsx`:
```javascript
{ to: '/new-route', label: 'New Route' }
```

3. Create feature folder: `src/features/NewFeature/`

### Adding a Blog Post

1. Add post object to `src/features/Blog/blogData.js`:
```javascript
{
  id: 'post-id',
  slug: 'post-slug',
  title: 'Post Title',
  date: 'DD Month YYYY',
  readTime: 'X min read',
  category: 'Category',
  excerpt: 'Brief description',
  content: '<p>HTML content...</p>'
}
```

### Adding Assets

- **Logos**: `src/assets/companies/` (prefer SVG format)
- **Project images**: `src/assets/projects/`
- **Other images**: `src/assets/` (organize by type)

### Styling Components

- Use Tailwind utility classes
- Follow design system patterns
- Use responsive classes: `max-[950px]:` for mobile
- Maintain consistent spacing: `gap-6`, `p-6`, etc.

## Code Style

- **Components**: Functional components with hooks
- **Naming**: PascalCase for components, camelCase for functions/variables
- **Imports**: Group by type (React, router, components, assets)
- **JSX**: Use semantic HTML, proper accessibility attributes
- **Comments**: Minimal, only when logic is complex

## Testing Changes

Before committing:

1. Run `npm run dev` and test locally
2. Check all routes work correctly
3. Test responsive design (mobile, tablet, desktop)
4. Verify no console errors
5. Ensure build succeeds: `npm run build`

## Git Workflow

- **Pre-commit**: Husky automatically runs `npm run build` and stages `docs/`
- **Commits**: Include both source code and built files
- **Branch**: Typically work on `master` branch (simple workflow)

## Common Tasks

### Adding a New Feature

1. Create feature folder: `src/features/NewFeature/`
2. Create components in feature folder
3. Add route to `src/App.jsx`
4. Add navigation link to `src/components/Header.jsx`
5. Update breadcrumbs if needed: `src/components/Breadcrumbs.jsx`

### Modifying Styles

1. Use Tailwind classes (preferred)
2. Check `KEVIN_DESIGN_SYSTEM.md` for patterns
3. Maintain responsive design
4. Test on multiple screen sizes

### Removing Code

1. Search for all references before deleting
2. Remove unused imports
3. Delete unused files
4. Update documentation if needed

## Important Files

- **`src/App.jsx`**: Main routing configuration
- **`src/components/Header.jsx`**: Navigation
- **`vite.config.js`**: Build configuration
- **`tailwind.config.js`**: Tailwind configuration
- **`package.json`**: Dependencies and scripts

## Deployment

- **Build output**: `docs/` folder
- **Auto-build**: Husky pre-commit hook
- **Deployment**: Automatic via GitHub Pages
- **Domain**: `zkevinbai.com` (configured in `public/CNAME`)

**Never manually edit files in `docs/`** - they are generated by the build process.

## When Making Changes

1. **Read relevant docs**: Check SYSTEM_DESIGN.md and KEVIN_DESIGN_SYSTEM.md
2. **Follow patterns**: Match existing code style and structure
3. **Test thoroughly**: Verify changes work across routes and screen sizes
4. **Clean up**: Remove unused code, imports, and files
5. **Update docs**: If architecture changes, update SYSTEM_DESIGN.md

## Questions to Ask

If unsure about something:

1. Is this consistent with existing patterns?
2. Does this follow the design system?
3. Will this break existing functionality?
4. Is there unused code I should remove?
5. Should I update documentation?

## Red Flags

⚠️ **Don't do these:**

- Add unnecessary dependencies
- Create complex state management (use local state)
- Add backend/server code (this is a static site)
- Break existing routes or navigation
- Ignore the design system
- Commit without testing the build
- Manually edit files in `docs/` folder
